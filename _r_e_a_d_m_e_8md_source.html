19 ### Description
20  The goal of this board is to have an easy-to-use, configurable GPS tracker that passes its data to an MQTT server.
21  It should also be able to scan other environmental data and operate on a 12V power supply with low power consumption
22  over a long period of time.
23 
24  The main components of the systems are:
25  * A 'Wemos D1 mini lite' to run the software.
26  * A BK-808 v2 (SIM808) GPS/GPRS module to communicate with an MQTT server and to send/receive SMS.
27  * A BME280 module to scan environmental data.
28  * An LM2596 DC-DC converter to supply the GPS/GPRS board.
29  * An MCP1703 for the 3.3V Wemos power supply.
30 
31  To interact with the board it has an easy-to-use browser interface to show the
32  scanned values, configure the system or to debug at runtime via console.
33  It can connect to an existing WiFi or function as an access point.
34 
35  The board has a voltage divider to measure the power supply voltage and
36  a BME280 sensor to read temperature, humidity and air pressure.
37 
38  The module operates with a supply voltage of approx. 6V to 16V. Perfect for a car battery.
39 
40  For power saving the supply voltage can be used to decide if the battery is charging or not
41  and so the system can be configured to go into deep sleep mode which uses only ~0.3mA.
42 
43  With the GSM chip, the system can receive SMS commands to set default values or send current
44  information such as GPS or temperature.
45 
46  The system estimates the current power consumption in mAh in energy saving mode and when the sim module is on.
47 
48  You can also use the software on a esp8266 chip like the wemos d1 for having a serial console over HTTP if you connect
49  to the D5 and D6 pin.

85 ### Setup Arduino Environment
86  * Download Arduino IDE from
87  https://www.arduino.cc/en/Main/Software
88  and follow the installation instructions.
89  * Install ESP8266 board software
90  * In Arduino IDE set 'File | Preferences' to Boards Manager URL
91  http://arduino.esp8266.com/stable/package_esp8266com_index.json
92  * Open 'Tools | Board... | Boards Manager...'
93  and search for 'esp8266' and install the software package
94  * Install the ESP8266 file-system up-loader from
95  https://github.com/esp8266/arduino-esp8266fs-plugin
96  and follow the installation instructions.
97  * Download **SnorkTracker** software by clone or download.
98  * Copy the content of the snorktracker/libraries directory to the Arduino installation libraries directory.
99  For example c:/Program Files (x86)/Arduino/libraries/
100  * Configure Arduino IDE
101  * From 'Tools | Boards' choose 'LOLIN(WEMOS) D1 mini Lite'
102  * Change 'Tools | Flash Size' to '1M (64k SPIFFS)'
103  * Change 'Tools | IwIP variant' to 'v2 Higher Bandwidth'

112 ### Assemble the Components
113  * Take the Breadboard and the other items from the part-list.
114  ![Components](images/Components.jpg "Components")
115  * First of all unsolder the (ON/OFF) pin 5 from the DC-DC module, bend it up and
116  solder a line to it. After that put 4 spacers under the DC-DC module and solder it
117  to the breadboard. Check that the bottom of the LM2596 module is not in contact with the board.
118  See
119  ![DC-DC module](images/DC-DC-Modul.jpg "DC-DC module")
120  * Place the other parts from the part-list on the breadboard and solder it as shown.
121  Pay attention to the polarity of the capacitors!
122  Split the header connector to the length that is needed and solder it.
123  A header connector is needed because flashing of the Wemos module often does not work
124  if it is still connected to the board.
125  Check if the Wemos and SIM808 module has enough space and fits into the pins.
126  ![Unplugged](images/Unplugged.jpg "Unplugged")
127  * Connect all the pins of the parts as seen on the circuit image with wires.
128  ![Back](images/Back.jpg "Back")
129  * The final board could look like this
130  ![Snorktracker board](images/SnorkTracker.jpg "Snorktracker board")
131  * Check all the wiring with a multimeter. Also against short-circuit.

133 ### Setup the LM2596 voltage and install the software
134  * The first step we have to do is to set the right 5V for the SIM808 module.
135  * For that unplug the Wemos and the SIM808 module.
136  * Connect a 12 power supply to the screw connector. Make sure + and - are correct!
137  * Switch on the LM2596 module by connecting the unsoldered pin 5 to ground.
138  * Now, connect the multimeter to the output pins of the DC-DC module and screw on
139  the module resistor until the multimeter shows exactly 5.0V on the two out-pins.
140  * Check the pins on the female header connector for the right voltage.
141  * Check the 3.3V on the corresponding WEMOS pin.
142  * Check the 5.0V pin of the SIM808 module connector pin.
143  * Check the 3.3V on the BME280 module. Be aware that the ground pin is connected to
144  the Wemos module and still open.
145  * Now its time to flash the software to the Wemos module.
146  * Unplug the Wemos module (if plugged) and connect it with the USB cable to your computer.
147  After connecting via usb you should be able to select the arduino port under 'Tools | Port | ...'.
148  * Open the tracker.ino file in the tracker folder and check the Arduino environment settings.
149  * Upload the SPIFFS files with the 'Tools | ESP8266 Sketch data upload' menu.
150  The SPIFFS files (htmls, styles and javascripts) are taken from the data subdirectory.
151  * After that go to the *Config.h* file of the project and enter the right configuration values.
152  * Now we can flash the program to the Wemos chip by clicking 'Sketch | upload'
153  * Unplug and plug again the Wemos module to the usb cable and start immediately the serial monitor
154  of the Arduino IDE 'Tools | Serial monitor'.
155  * If everything is correct you can see an access-point in the WiFi of your computer and you can connect
156  to the browser interface by calling the ip in your browser.
157  The software works in a captive way, so you can enter a dummy (not https) url and the system should redirect you
158  to the web interface. For example http://dummy.com
159  If this does not work find out the new ip via your WiFi router interface.
160  * If the software works fine on the usb cable then you can plug the wemos into the board and connect a power supply.
161  Check again the connectivity.

163 ### Function
164  * The software in the Wemos module creates a web-server in station and/or in access-point mode.
165  * The analog input of the wemos is connected to the middle of the two voltage divider resistors.
166  So it can read the power supply voltage of the system (for example the car battery voltage).
167  * The software can be configured to recognize a charging or discharging car battery to switch on or off the
168  submodules. With my car this was:
169  * 11.8 Volt if the car was switched off for more than a day.
170  * 14.2 Volt if the engine is running.
171  * 13.3 Volt after switching off the engine.
172  * A BME280 sensor is connected to the 3.3V power and can be switched on by setting the D4 pin of the wemos chip
173  to ground. This is done only from time to time to save energy.
174  * There is an LM2596 DC-DC module on the board which can be switched on by a '-' signal to the pin 5 of the LM2596.
175  Behind the DC-DC module there is a SIM808 module with GPS/GPRS/GSM functionality. So the Wemos can switch on/off
176  the SIM808 chip to save energy.
177  * The Wemos chip can communicate with the SIM808 module via RX and TX signal and AT commands.
178  * Via the GPRS module it can send the scanned data to a MQTT server and can communicate via SMS to a phone.
