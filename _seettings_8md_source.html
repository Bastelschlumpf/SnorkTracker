2 ## Overview of the system settings
3 
4 ### Connect to the Web Interface
5 The easiest method is when the module automatically logs into your Wifi network.
6 To do this you change the two defines **WIFI_SID** and **WIFI_PW** in Config.h before compiling.
7 If you supply then the system with power, it starts the integrated web server and logs into
8 your WiFi network. You can use your router software to search for the new SnorkTracker IP.
9 You can then open a web browser and enter this new ip as http://xxx.xxx.xxx.xxx
10 Attention: The system does not support https yet so it only works if the URL starts with **http:**
11 
12 If you have problems finding out the new IP, you can also simply search for the new access point
13 called **SnorkTracker** in your WiFi settings and then connect to it.
14 If you have connected to this access point you can now connect with a browser.
15 Since the system works with a captive portal and redirects all unknown urls to its own,
16 it is sufficient if you enter a random http: - url.
17 e.g. http://Dummy.html
18 Make sure you use http and not https.

35 ### GPRS Settings
36 ![GPRS Settings](../images/SettingsGprs.png "GPRS Settings")
37 
38 Be sure that you have inserted a valid SIM card with a valid contract into the SIM card slot
39 of the SIM808. Note that the system connects without a password.
40 So the easiest way to check if the system works without a password and to check if it can connect
41 to a GPRS network is to put it into a mobile phone and check on start for password query and
42 browsing functionality.
43 
44 To connect to the Internet different gprs provider use different gprs access point names.
45 For example, sunrise uses the access point name 'INTERNET' and swisscom the name 'gprs.swisscom.ch'.
46 Search the Internet for the access point name of your provider.

48 ### Enable debugging
49 ![Debug Settings](../images/SettingsDebug.png "Debug Settings")
50 
51 The system has possibility to put more detailed information to the console window.
52 If you want to know i.e. the detailed communication between the esp8266 and the sim808 module
53 you can switch on here the display of this information.
54 For performance reason it is better to switch off this if it is not needed.

56 ### Enable environment sensor
57 ![BME280 Settings](../images/SettingsBME280.png "BME280 Settings")
58 
59 The system is connected to a BME280 sensor. This sensor can scan the temperature, humidity and
60 the pressure of its environment.
61 With this settings you can control the interval of the scan. These environment values doe not change so
62 often so it is enough if this interval is set to a bigger value like 1 minute or bigger.
63 
64 ### SMS Settings
65 ![SMS Settings](../images/SettingsSms.png "SMS Settings")
66 
67 If the system is switched on and it has a valid gprs connection details of it can be controlled
68 via incoming sms.
69 Here you can set the interval of this sms check.

71 ### GPS Settings
72 ![GPS Settings](../images/SettingsGps.png "GPS Settings")
73 
74 If the system is switched on you can also switch on the gps check functionality and the
75 gps check interval here.
76 Remember that it can take up to a minute for the sim808 module to receive a valid gps signal.
77 If you configure a gps check interval below this the system waits until it has a valid gps value.
78 
79 ### MQTT Settings
80 ![MQTT Settings](../images/SettingsMqtt.png "MQTT Settings")
81 
82 With this settings part you can configure the mqtt connection to your mqtt server.
83 If the system has a valid mqtt server connection it sends only the information that are valid
84 at this moment. So, for example if you switched off the gps then this information could not be
85 send to your server.
86 If you have switched on the gps sending than you can additionally configure here how often the gps
87 values should be send while your system is moving and how often if the system stands fix on one place.
88

92 ![MQTT Server Settings](../images/MqttServer.png "MQTT Server Settings")
93 
94 Remember the system works with HTTP and not with HTTPS. So be sure you can connect with this
95 protocol to your MQTT server.

97 ### Power Save Setting
98 ![Deep Sleep Settings](../images/SettingsDeepSleep.png "Deep Sleep Settings")
99 
100 If you connect the system to a battery it is important that you don't use to much power consumption.
101 Here you can configure that the system can detect if your battery is powered via a generator or not.
102 Make some tests how much voltage the battery has when charging and when not charging.
103 Then set a value between the two values here so the system can sleep if not charging.
104 It is better to set the active time to low value because the system waits with the sleep mode
105 if all the needed operations (gps, mqtt, sms) are done.
